## 両利き経営

### 概要
既存事業を磨き上げる「深化」と、新規事業やイノベーションに挑戦する「探索」を同時に追求する経営スタイル（Organizational Ambidexterity）。

### 深化と探索

| 活動 | 内容 |
|------|------|
| 深化（Exploitation） | 既存事業の効率化・品質向上・コスト削減・顧客満足向上 |
| 探索（Exploration） | 新市場・新技術・新ビジネスモデルへの挑戦、PoC、スタートアップ連携 |

### 両利き経営が重要な理由

- 既存事業依存→「成功の罠」で環境変化への適応が遅れる
- 探索偏重→キャッシュフロー弱化で新規事業への投資継続が困難
- 深化が投資原資、探索が将来の成長エンジンとなり、両立で高成長と業績安定を実現

### 組織デザインのパターン

| パターン | 内容 |
|----------|------|
| 構造的分離（ストラクチュラル型） | 既存事業と新規事業を組織として分け、異なるKPI・評価・文化を持たせる |
| 文脈的両利き（コンテクスチュアル型） | 各部門・個人レベルで日々の仕事に深化と探索を両方織り込む |
| ハブ・アンド・スポーク型 | 経営者がハブとなり各事業ユニットをコントロール |

### 実践のポイント

- **戦略的意図とビジョンの明確化**：なぜ探索が必要か、既存との両立方針を共有
- **トップマネジメントのコミットメント**：資源配分・評価制度で支援
- **別々だが連携したアーキテクチャ**：探索には自由度・実験志向、深化には効率性・安定性
- **組織文化・人材の整備**：失敗許容文化、学習する組織

### 中小企業への示唆

- 既存事業のキャッシュ創出力を高めつつ、一部を新規事業の実験に投じる「ポートフォリオ思考」
- 専任探索部門を作れない場合、既存事業に近いadjacent領域から小さなPoCを繰り返す「軽い両利き」

## 新規事業化のポイント

### 全体設計思想

- 新規事業がうまくいかない主因はアイデアの質より「既存事業の成功ゆえの抵抗（企業の免疫）」
- 既存は効率・確実性、新規は試行錯誤・失敗許容が前提→同じルール・評価軸で運営すると新規が窒息
- **出島的な独立ユニット＋既存との橋渡し役**を設計

### 最初に決めておくべきこと

| 項目 | 内容 |
|------|------|
| マイルストン | 何年目で何をもって続行/縮小/撤退を判断するか |
| 権限と予算 | 新規事業側の意思決定権限と予算枠（既存と同じ稟議にしない） |
| シナジー設計 | 既存事業とどこで共通資産（人材・顧客・ブランド）を活用するか |

### 社内抵抗の構造

| 理由 | 内容 |
|------|------|
| 自己的利益の喪失 | ポジション・権限・専門性の陳腐化への恐れ |
| 経営への不信感 | 「どうせ途中でやめる」「責任だけ押し付けられる」 |
| 必要性の認識差 | 「今でも食えてるのに、なぜやるのか」 |
| 能力・耐性の問題 | 変化に弱い、不安や負荷が大きい |

### 古参社員・既存部門への対処

1. **存在価値を守る**
   - 「自分の居場所・優位性がある」感覚を維持
   - これまでの貢献・現場知を認めつつ、新体制・新事業の方針を経営者自ら説明
   - 古参を「守りの責任者」「品質ゲート」「後進育成」に位置づけ

2. **抵抗勢力を取り込む**
   - 強い抵抗が予想される人ほど、早い段階から計画立案・推進チームに参加させる
   - 「蚊帳の外に置かれた」感覚が反発を強める

3. **合わない場合の対処**
   - 社長が正面から「これから会社をこうしていきたい」と伝え、協力の可否を確認
   - ラインから外してプロジェクト型・顧問的な位置づけに変更

### 新規事業側の体制・評価

| 観点 | 内容 |
|------|------|
| 置き場所・権限 | 社長直下に置き、既存部門からの雑音に負けないメッセージ |
| 人材配置 | エース級人材を投入（「余り人材でやる」は失敗要因） |
| 評価指標 | 短期売上より「仮説検証サイクルの数・質」「顧客インサイト獲得」 |
| 協力評価 | 既存部門が新規に協力した場合、その貢献も人事評価に反映 |

### 新規と既存の橋渡し

- 共通資産（ブランド・顧客基盤・インフラ・人材）の共有ルールを事前策定
- 両事業の言語がわかる橋渡し役を配置

## リソース戦略（コアとBPO）

### コア業務とノンコア業務

| 分類 | 内容 |
|------|------|
| コア業務 | ミッション・ビジネスモデルに直結、売上・利益・競争優位に寄与、ノウハウが蓄積される領域 |
| ノンコア業務 | 事業運営には必要だが差別化要因になりにくい、汎用的・定型的な業務 |

**重要**: 「社員＝コア、外注＝ノンコア」ではなく、「業務がコアかどうか」でまず分類し、その上で社員か外注かを決める

### 社員（コア人材）に任せる領域

| 観点 | 例 |
|------|-----|
| 戦略的重要度が高い | 方向性・事業ポートフォリオ・プロダクト戦略・重要パートナーシップ設計 |
| 競争優位の源泉 | 独自アルゴリズム・プロダクト開発、主要顧客向けソリューション設計 |
| ノウハウを蓄積したい | 将来も自社のコアとして維持・進化させたい領域 |

IT/コンサル系の例：「要件定義〜アーキ設計」「業界知見を生かした提案」「顧客折衝」は社員中心

### 外注・BPOに任せる領域

| 観点 | 例 |
|------|-----|
| 定型化しやすい | 給与計算、経理処理、標準的な問い合わせ対応、データ入力 |
| 高度だが汎用的な専門性 | 法務、税務・会計、標準的なシステム運用保守 |
| スケール・変動対応 | コールセンター、EC出荷・物流、キャンペーン時の急増対応 |

**メリット**: コア業務への集中、コスト削減、スピード向上
**リスク**: ノウハウ蓄積しにくい、過度依存でコントロール低下

### 判断軸

| 軸 | 社員が向く | 外注・BPOが向く |
|-----|-----------|---------------|
| 戦略的重要度 | 高い（事業の核、将来の柱） | 中〜低い（代替可能、インフラ的） |
| 競争優位との関係 | 差別化要因になりうる | 差別化になりにくい標準業務 |
| ノウハウ蓄積 | 高い、長期的に自前で育てたい | 低い、外部ノウハウ活用で十分 |
| ボリューム・平準性 | 継続的に一定以上 | 変動大きい、期間限定、季節要因 |
| 品質・機密性要求 | 高い統制・機密保持が必要 | 標準品質で足り、機密性も許容範囲 |

### 棲み分け設計のステップ

1. **全業務を棚卸し**：部署・プロセス単位で目的・成果・頻度・必要スキルを整理
2. **コア／ノンコア分類**：ミッション貢献、競争優位との関係、自前で持つ必要性を基準に
3. **社員が担う中核と外部化候補を決定**：コアでも定型部分だけBPOに出す分割も検討
4. **外部化のメリット・リスク評価**：コスト、品質、スピード、ナレッジロス、依存リスク
5. **パイロット導入と標準化**：1プロセス・1部署で試し、手順書・SLAを整備してスケール

### 設計図のイメージ

「①コア業務の定義」と「②社員の役割（上流・設計・マネジメント）」を固め、その下に外注・BPOを配置する組織設計図を描く
