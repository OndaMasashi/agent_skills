# 🛠️ スキル利用ガイド (Skill Catalog)

現在あなたの環境で利用可能なスキルのカタログです。
エージェントに対して、以下の「呼び出しキーワード」を含む指示を出すことで、各スキルを自動的に活用します。

---

## 📄 ドキュメント関連 (Document Skills)

文書の作成、編集、分析、変換などを行うスキルです。

| スキル名 | 呼び出しキーワード例 | 詳細な機能・特徴 |
| :--- | :--- | :--- |
| **Word (DOCX)** | 「Wordを作成」「レポートをdocxで」 | スタイル設定、画像の挿入、構成案からの自動生成、既存文書の校正とリライト。 |
| **Excel (XLSX)** | 「Excelを作って」「表計算ファイルを」 | 複雑な計算式の埋め込み、集計テーブルの作成、CSVデータからの表変換。 |
| **PowerPoint (PPTX)** | 「スライドを作って」「pptxでプレゼン案」 | 構成案に基づくスライドの自動生成、レイアウト設計、発表用ノートの作成。 |
| **PDF** | 「PDFを結合」「PDFからテキスト抽出」 | 複数ファイルの結合・分割、スキャン済みPDFのテキスト化、特定ページの抽出。 |
| **Word文字数カウント** | 「docxとしての文字数をカウント」 | Markdown形式からWord(A4/40字×40行)に書き出した際の想定文字数・枚数を精密に推定。 |
| **Google Workspace** | 「Googleドキュメントを」「Driveから」 | ブラウザを介さず直接ドキュメント作成やDrive内のファイル検索・読み込みを実行。 |
| **ドキュメント自動保存** | 「計画をバックアップ」「walkthrough保存」 | 重要な計画（implementation_plan）や成果報告（walkthrough）を自動で指定フォルダにバックアップ。 |
| **アウトライン** | 「要約して」「アウトラインを作成」 | 長文の論理構造を抽出し、見出しレベルに応じた階層的な構成案を生成。 |

## 💻 開発・技術関連 (Development Skills)

プログラミング、デプロイ、テスト、設計などを支援するスキルです。

| スキル名 | 呼び出しキーワード例 | 詳細な機能・特徴 |
| :--- | :--- | :--- |
| **Vercel Deploy** | 「デプロイして」「URLを共有して」 | ローカルプロジェクトを瞬時に公開。プレビューURLに加えて、後で自分のアカウントに紐付けられるClaimリンクを発行。 |
| **React/Expo開発** | 「React Nativeで作って」「Expoで」 | iOS/Android両対応のモバイルアプリおよびWebアプリのコード生成、画面遷移、コンポーネント実装。 |
| **Reactベストプラクティス** | 「Reactの設計方針は？」「フックの使い方」 | 依存配列の最適化、状態管理の選定、パフォーマンスを考慮した最新のコーディング標準に基づく助言。 |
| **設計パターン** | 「コンポーネント構成案」「Render Props」 | 高階コンポーネント(HOC)やCompositionを用いた、DRYでメンテナンス性の高いコンポーネントツリーの設計。 |
| **Find Skills** | 「スキルを検索」「どんなスキルがある？」 | ユーザーの漠然とした要望から最適なスキルを特定し、その使い方やメリットをガイド。 |
| **Skill Creator** | 「新しいスキルを作成して」「カスタムスキル」 | 独自の定型業務を自動化するためのスキル（指示書、スクリプト、テンプレート）を設計・パッケージ化。 |
| **MCP Builder** | 「MCPサーバーを作成」「MCPツールを」 | 外部APIや独自DBをエージェントに接続するための次世代プロトコル（Model Context Protocol）の実装支援。 |
| **Webアプリテスト** | 「UIテストを実行」「E2Eテストをして」 | ブラウザを自動操作し、ボタンクリックやフォーム入力などのユーザーフローが正常か検証。 |
| **Deep Research** | 「詳細に調査して」「多角的に調べて」 | 表面的な検索を超え、複数の情報源を横断して競合分析や技術トレンドの深掘り調査を実施。 |
| **データベース (Postgres)** | 「SQLを実行」「DBを操作」 | スキーマ設計の提案から複雑なSELECT句、ジョインを含むクエリの作成と実行、デバッグ。 |

## 💡 使い方のコツ

- **複数を組み合わせる**: 「調査スキルで調べて、Wordスキルでレポートにして」といった複合的な指示も可能です。
- **具体的に指示する**: 指示内容に具体的な目的を入れると、エージェントがより最適なスキルを選択します。
- **わからない時は聞く**: 「〇〇ができるスキルはある？」と聞けば、エージェントが最適なものを提案します。
